What I use.

工具清单

A living document of the hardware, AI tools, notes, and basic tech stack I use while learning product through prototypes.

这是一份实时更新的清单，记录我在通过原型学习产品时使用的设备、AI 工具、笔记工具和基础技术栈。

Hardware

硬件设备

Windows Laptop Daily Driver

My main machine for research, writing, AI-assisted prototyping, and local project work.

我的主力设备，用来做研究、写东西、用 AI 辅助做原型，以及处理本地项目。

iPad Reading

I use it for reading, sketching rough flows, and collecting references away from the laptop.

我用它阅读资料、简单画流程，也在离开电脑时收集一些参考。

External Monitor Workspace

A larger workspace helps when comparing references, writing notes, and keeping a prototype open beside source material.

更大的工作空间适合对比参考、整理笔记，也方便把原型和资料并排打开。

Software & Editor

软件与编辑器

Codex Agent

My main coding agent for turning rough ideas, page drafts, and product experiments into working prototypes.

我常用的编码 Agent，用来把粗略想法、页面草稿和产品实验推进成可以运行的原型。

Obsidian Notes

Where I keep product notes, learning records, references, and early ideas before they become prototypes.

我用它存放产品笔记、学习记录、参考资料，以及变成原型之前的早期想法。

Open Design Design

A visual companion for checking designs, marking changes, and keeping the prototype close to the intended look.

我用它检查设计、标注修改点，并尽量让原型接近想要的视觉效果。

Claude Code Coding

Another AI coding tool I use to explore implementations, compare approaches, and learn from generated code.

另一个我会使用的 AI 编码工具，用来探索实现方式、比较方案，也从生成的代码里学习。

Tech Stack

技术栈

Python Language

Python is the language I know best right now. I use it as a base for understanding logic, scripts, and AI-related experiments.

Python 是我目前最熟悉的语言。我用它作为理解逻辑、脚本和 AI 相关实验的基础。

JavaScript Frontend

I'm learning enough JavaScript to understand interactive pages, browser extensions, and prototype behavior.

我正在学习足够的 JavaScript，用来理解交互页面、浏览器扩展和原型行为。

React UI

React helps me understand how interfaces are structured and how product ideas can become reusable UI pieces.

React 帮助我理解界面如何被组织，也理解产品想法如何变成可复用的 UI 片段。

Next.js Framework

I'm learning enough frontend to turn ideas into usable prototypes and to understand how modern web products are put together.

我也在学习足够的前端知识，把想法做成可以试用的原型，并理解现代 Web 产品是如何组织起来的。
