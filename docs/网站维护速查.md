# 网站维护速查

这份文档只写最常用的修改位置。下面所有路径都从项目文件夹 `personal web/` 开始。

## 1. 本地预览

打开项目文件夹后运行：

```bash
npm run dev
```

浏览器打开：

```text
http://localhost:3000/
```

## 2. 改完上线前先跑

```bash
npm test
npm run build
```

两个都没报错，再提交和推送：

```bash
git status -sb
git add .
git commit -m "你的修改说明"
git push origin main
```

推送后 GitHub Pages 会自动部署。线上地址是：

```text
https://musu6856.github.io/Personal-Web/
```

注意：`https://musu6856.github.io/` 单独打开 404 是正常的，这个网站在 `/Personal-Web/` 下面。

## 7. 图片替换规则

最省事的方式：

1. 新图片放到原来的位置。
2. 新图片改成原来的文件名。
3. 后缀也保持一样，比如原来是 `.png` 就继续用 `.png`。

这样不用改代码。

如果文件名或后缀变了，去这里改路径：

```text
personal web/content/assets.ts
```

网站里写的路径不带 `public/`。例如文件在：

```text
personal web/public/assets/site/about.png
```

代码里写的是：

```text
assets/site/about.png
```

## 11. 新增一篇博客

基本要改两个地方：

1. 在 `personal web/content/posts.ts` 里加博客信息。
2. 在 `personal web/lib/site-renderers.ts` 里加对应正文。

图片放这里：

```text
personal web/public/assets/posts/你的博客-slug/cover.png
personal web/public/assets/posts/你的博客-slug/body-figure.png
```

再去 `personal web/content/assets.ts` 登记图片路径。

## 12. 新增一个项目

基本要改三个地方：

1. 在 `personal web/content/projects.ts` 里加项目卡片信息。
2. 在 `personal web/content/project-details.ts` 里加详情页文案。
3. 在 `personal web/content/assets.ts` 里登记图片路径。

图片建议放这里：

```text
personal web/public/assets/projects/你的项目-slug/card.png
personal web/public/assets/projects/你的项目-slug/cover.png
```

如果暂时不想显示，把 `published` 设为 `false`。

## 13. 常见问题

### 本地图片不显示

先检查三件事：

1. 图片是不是真的在 `personal web/public/assets/...` 下面。
2. 文件名和后缀有没有写错。
3. `personal web/content/assets.ts` 里的路径有没有改对。

### 线上改了但没变化

先等 GitHub Actions 部署完成，再强制刷新浏览器。

### 页面跳转 404

线上地址要带 `/Personal-Web/`。如果你复制的是根地址，很容易 404。

### 图片太大、太小、裁切不满意

这通常不是图片文件的问题，是页面样式的问题。先不要乱改图片路径，直接改对应页面的 CSS。
